\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Prim\+Type\+\_\+$>$\newline
class kindr\+::\+Euler\+Angles\+Zyx$<$ Prim\+Type\+\_\+ $>$}

Implementation of Euler angles (intrinsic Z-\/Y\textquotesingle{}-\/X\textquotesingle{}\textquotesingle{}, a.\+k.\+a. extrinsic x-\/y-\/z, a.\+k.\+a. yaw-\/pitch-\/roll angles) rotation. 

A vector (yaw, pitch, roll) of intrinsic Z-\/Y\textquotesingle{}-\/X\textquotesingle{}\textquotesingle{} Euler angles represents the following sequence of rotations from child to parent\+: \begin{DoxyVerb}rotationChildToParent = R_z(yaw) R_y(pitch) R_x(roll)
                      = [ cos(yaw)  -sin(yaw)  0 ] [ cos(pitch)   0  sin(pitch) ] [ 1  0         0          ]
                        [ sin(yaw)  cos(yaw)   0 ] [ 0            1  0          ] [ 0  cos(roll) -sin(roll) ]
                        [ 0         0          1 ] [ -sin(pitch)  0  cos(pitch) ] [ 0  sin(roll) cos(roll)  ]
\end{DoxyVerb}


where R\+\_\+k(angle) is the rotation matrix of a positive rotation of {\ttfamily angle} around the k-\/axis. Starting from the parent frame and applying active rotations, we first rotate the frame x-\/y-\/z by {\ttfamily yaw} around the z-\/axis to obtain a frame x\textquotesingle{}-\/y\textquotesingle{}-\/z\textquotesingle{}, then by {\ttfamily pitch} around the y\textquotesingle{}-\/axis to obtain x\textquotesingle{}\textquotesingle{}-\/y\textquotesingle{}\textquotesingle{}-\/z\textquotesingle{}\textquotesingle{}, and finally by {\ttfamily roll} around the x\textquotesingle{}\textquotesingle{}-\/axis to obtain the child frame x\textquotesingle{}\textquotesingle{}\textquotesingle{}-\/y\textquotesingle{}\textquotesingle{}\textquotesingle{}-\/z\textquotesingle{}\textquotesingle{}\textquotesingle{}.

\begin{DoxyNote}{Note}
This definition by intrinsic Z-\/Y\textquotesingle{}-\/X\textquotesingle{}\textquotesingle{} rotations is equivalent to (i.\+e. yields the same rotation matrix as) the definition by extrinsic x-\/y-\/z rotations where rotations are applied in the reverse order and using the fixed axes of the parent frame. This matches the convention used in the U\+R\+DF documentation \href{https://wiki.ros.org/urdf/XML/joint}{\texttt{ https\+://wiki.\+ros.\+org/urdf/\+X\+M\+L/joint}}\+: \char`\"{}rotation around fixed axis\+: first roll around x,
then pitch around y and finally yaw around z\char`\"{}.
\end{DoxyNote}
