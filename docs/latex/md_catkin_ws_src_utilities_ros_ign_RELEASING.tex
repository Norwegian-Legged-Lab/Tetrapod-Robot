Different branches of this repository support different combinations of R\+OS 1, R\+OS 2 and Ignition. Refer to the R\+E\+A\+D\+ME to see what combinations are supported.

We use 2 separate approaches for releasing R\+O\+S-\/enabled binaries\+:


\begin{DoxyItemize}
\item Release upstream into \href{https://packages.ros.org,}{\texttt{ https\+://packages.\+ros.\+org,}} so it\textquotesingle{}s easily available to all R\+OS users.
\item Release into \href{https://packages.osrfoundation.org,}{\texttt{ https\+://packages.\+osrfoundation.\+org,}} which requires extra setup for R\+OS users.
\end{DoxyItemize}

Releasing into R\+OS packages is ideal, but that\textquotesingle{}s only possible if the necessary Ignition libraries are available either through official Ubuntu packages, or directly from \href{https://packages.ros.org}{\texttt{ https\+://packages.\+ros.\+org}}. This situation varies according to the Ignition version\+:


\begin{DoxyItemize}
\item Blueprint and Dome are only available from \href{https://packages.osrfoundation.org}{\texttt{ https\+://packages.\+osrfoundation.\+org}}
\item Citadel is available from\+:
\begin{DoxyItemize}
\item \href{https://packages.osrfoundation.org:}{\texttt{ https\+://packages.\+osrfoundation.\+org\+:}} all Ubuntu versions
\item \href{https://packages.ros.org:}{\texttt{ https\+://packages.\+ros.\+org\+:}} only Ubuntu Focal
\end{DoxyItemize}
\end{DoxyItemize}

Another factor to take into consideration is which Ignition version is officially supported for each R\+OS distro according to the following R\+E\+PS\+:


\begin{DoxyItemize}
\item R\+OS 1\+: \href{https://ros.org/reps/rep-0003.html}{\texttt{ R\+E\+P-\/0003}}
\item R\+OS 2\+: \href{https://www.ros.org/reps/rep-2000.html}{\texttt{ R\+E\+P-\/2000}}
\end{DoxyItemize}
