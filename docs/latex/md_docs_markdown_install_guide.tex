This page contains relevant information about installing the dependencies needed for building and running the R\+OS Environment found in \href{../../catkin_ws/}{\texttt{ catkin\+\_\+ws}}.\hypertarget{md_docs_markdown_install_guide_autotoc_md18}{}\doxysection{R\+OS}\label{md_docs_markdown_install_guide_autotoc_md18}
To install R\+OS Noetic\+:


\begin{DoxyItemize}
\item First setup your computer to accept packages from packages.\+ros.\+org\+:
\end{DoxyItemize}


\begin{DoxyCode}{0}
\DoxyCodeLine{sudo sh -\/c 'echo "deb http://packages.ros.org/ros/ubuntu \$(lsb\_release -\/sc) main" > /etc/apt/sources.list.d/ros-\/latest.list'}
\end{DoxyCode}



\begin{DoxyItemize}
\item Setup your keys\+:
\end{DoxyItemize}


\begin{DoxyCode}{0}
\DoxyCodeLine{curl -\/s https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc | sudo apt-\/key add -\/}
\end{DoxyCode}



\begin{DoxyItemize}
\item Installation\+:
\end{DoxyItemize}


\begin{DoxyCode}{0}
\DoxyCodeLine{sudo apt update}
\DoxyCodeLine{sudo apt install ros-\/noetic-\/desktop-\/full}
\end{DoxyCode}


To automatically source the necessary bash script, run the following commands\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{echo "source /opt/ros/noetic/setup.bash" >> \string~/.bashrc}
\DoxyCodeLine{source \string~/.bashrc}
\end{DoxyCode}



\begin{DoxyItemize}
\item Install and initialize Rosdep
\end{DoxyItemize}

rosdep enables you to easily install system dependencies for source you want to compile and is required to run some core components in R\+OS. rosdep can be installed and initialized with the following commands\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{sudo apt install python3-\/rosdep -\/y}
\DoxyCodeLine{sudo rosdep init}
\DoxyCodeLine{rosdep update}
\end{DoxyCode}

The Gazebo Simulator depends a number of environment variables to locate files, and set up communications between the server and clients and are explained in detail \href{http://gazebosim.org/tutorials?tut=components&cat=get_started}{\texttt{ here}}. For the current simulator iteration two enviroment variables are of interest, namely

{\ttfamily G\+A\+Z\+E\+B\+O\+\_\+\+M\+O\+D\+E\+L\+\_\+\+P\+A\+TH}\+: A colon-\/separated set of directories where Gazebo will search for models.

{\ttfamily G\+A\+Z\+E\+B\+O\+\_\+\+R\+E\+S\+O\+U\+R\+C\+E\+\_\+\+P\+A\+TH}\+: A colon-\/separated set of directories where Gazebo will search for other resources such as world and media files.

Assuming the \href{https://github.com/Pathorse/Tetrapod-Robot}{\texttt{ github repository}} is located at your home folder, i.\+e., at {\ttfamily $\sim$/}, the following set of lines can be added to set the needed enviroment variables.


\begin{DoxyCode}{0}
\DoxyCodeLine{\#-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/}
\DoxyCodeLine{\# Gazebo Setup}
\DoxyCodeLine{\#-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/}
\DoxyCodeLine{source /usr/share/gazebo/setup.sh}
\DoxyCodeLine{}
\DoxyCodeLine{export GAZEBO\_MODEL\_PATH=\string~/Tetrapod-\/Robot/catkin\_ws/src/simulator}
