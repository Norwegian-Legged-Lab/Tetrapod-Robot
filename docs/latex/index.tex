{\bfseries{joy}} R\+OS joystick driver for all linux joysticks. The driver will poll a given port until it can read from it, the publish Joy messages of the joystick state. If the port closes, or it reads an error, it will reopen the port. All axes are in the range \mbox{[}-\/1, 1\mbox{]}, and all buttons are 0 (off) or 1 (on).

Since the driver will poll for the joystick port, and automatically reopen the port if it\textquotesingle{}s closed, the joy\+\_\+node should be \char`\"{}on\char`\"{} whenever possible. It is typically part of the robot\textquotesingle{}s launch file.\hypertarget{index_autorepeat}{}\doxysubsubsection{Auto-\/\+Repeat/\+Signal Loss}\label{index_autorepeat}
The joy\+\_\+node takes an \char`\"{}$\sim$autorepeat\+\_\+rate\char`\"{} parameter. If the linux kernal receives no events during the autorepeat interval, it will automatically repeat the last value of the joystick. This is an important safety feature, and allows users to recover from a joystick that has timed out.\hypertarget{index_usage}{}\doxysubsubsection{Usage}\label{index_usage}
