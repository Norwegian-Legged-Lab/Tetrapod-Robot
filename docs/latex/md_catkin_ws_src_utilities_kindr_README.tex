\hypertarget{md_catkin_ws_src_utilities_kindr_README_autotoc_md75}{}\doxysubsection{Kindr 1.\+0.\+0}\label{md_catkin_ws_src_utilities_kindr_README_autotoc_md75}

\begin{DoxyItemize}
\item Simplified header include {\ttfamily \#include $<$kindr/\+Core$>$} is provided.
\item Kindr is now strongly dependent on Eigen.
\item All sub namespaces have been removed. (e.\+g. {\ttfamily kindr\+::rotations\+::eigen\+\_\+impl} -\/$>$ {\ttfamily kindr\+:\+:})
\item The implementations of rotations and time derivatives have been simplified (Passive, Hamiltonian).
\begin{DoxyItemize}
\item Active typedefs (e.\+g. Rotation\+Quaternion\+AD) have been removed and simpler ones (e.\+g. Rotation\+QuaternionD) have been introduced.
\item Note that the functionality of some operators changed! Please check the \href{https://opensource.docs.anymal.com/doxygen/kindr/master/cheatsheet_latest.pdf}{\texttt{ cheat sheet}} to understand what is implemented.
\item Some hints on what needs to be changed from kindr 0.\+0.\+1\+:
\begin{DoxyItemize}
\item {\ttfamily rotation.\+set\+From\+Vectors(v1, v2)} -\/$>$ {\ttfamily rotation.\+set\+From\+Vectors(v2, v1)}
\item {\ttfamily C\+\_\+\+B\+I.\+box\+Plus(dt $\ast$ B\+\_\+w\+\_\+\+I\+B)} -\/$>$ {\ttfamily C\+\_\+\+B\+I.\+box\+Plus(dt $\ast$ C\+\_\+\+I\+B $\ast$ B\+\_\+w\+\_\+\+I\+B)}
\item {\ttfamily C\+\_\+\+B\+I.\+box\+Minus(dt $\ast$  B\+\_\+w\+\_\+\+I\+B)} -\/$>$ {\ttfamily -\/\+C\+\_\+\+B\+I.\+box\+Minus(dt $\ast$ B\+\_\+w\+\_\+\+IB)}
\item Euler angles probably have to be negated.
\end{DoxyItemize}
\end{DoxyItemize}
\item Conversion methods between R\+OS and kindr have been moved to the package \href{https://github.com/anybotics/kindr_ros}{\texttt{ kindr\+\_\+ros}}.
\item Concatenation of Homogeneous Transformation is now implemented.
\item Short typedefs are provided for Homogeneous Transformation\+: {\ttfamily Hom\+Transform\+QuatD}, {\ttfamily Hom\+Transform\+MatrixD}.
\item Jacobian of exponential map is implemented.
\item Unit tests based on gtest are provided to test the convention of other software packages.
\begin{DoxyItemize}
\item Gazebo (gazebo\+::math\+::\+Quaternion) uses the same convention as kindr.
\item R\+OS TF (tf\+::\+Quaternion and tf\+::\+Matrix3x3) uses the same convention as kindr.
\item R\+B\+DL\textquotesingle{}s Rigid\+Body\+Dynamics\+::\+Math\+::\+Spatial\+Transform uses the same convention as kindr, whereas R\+B\+DL\textquotesingle{}s Rigid\+Body\+Dynamics\+::\+Math\+::\+Quaternion concatenates differently and its conversion to a rotation matrix is inverted.
\end{DoxyItemize}
